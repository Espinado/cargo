# Руководство администратора — Cargo TMS

Подробная инструкция по работе с веб‑приложением **Cargo TMS**: все разделы, доработки и основные действия для пользователя с полным доступом (администратор/менеджер).

---

## 1. Вход и навигация

- Вход в систему выполняется по **email и паролю** (учётная запись создаётся администратором).
- После входа открывается **главная страница (Dashboard)** — таблица истекающих документов.
- Слева расположено **боковое меню**:
  - **Dashboard** — сводка по истекающим документам (водители, тягачи, прицепы).
  - **Транспорт** — подменю: **Водители**, **Тягачи**, **Прицепы**, **Перевозчики** (сторонние), **Карта**, **Обслуживание**.
  - **Рейсы и заказы** — подменю: **Заказы**, **Рейсы**.
  - **Клиенты** — база клиентов (компании).
  - **Статистика** — подменю: **Дашборд владельца**, **Обзор** (рейсы за период), **События одометра**, **Клиенты**, **Простой**.
  - **Счета** — список счетов и платежи.
- В шапке: заголовок страницы и кнопка **Выйти**.

На мобильных устройствах меню открывается по кнопке «гамбургер» (☰); интерфейс адаптирован под мобильные экраны.

---

## 2. Dashboard (главная)

На главной странице (**/dashboard**) отображается **таблица истекающих документов** (срок действия в течение ближайших 30 дней).

- **Типы сущностей**: водители, тягачи, прицепы.
- **Документы**:
  - у **водителей**: права (License), Code 95, разрешение (Permit), медсправка (Medical), декларация (Declaration), медосмотр;
  - у **тягачей**: техосмотр (Inspection), страховка (Insurance), техпаспорт (Tech passport);
  - у **прицепов**: техосмотр, страховка, TIR, техпаспорт.
- Доступны **поиск**, **сортировка** по столбцам и **количество строк** на странице.
- По каждой строке видно: тип сущности, название (ФИО водителя / марка и госномер ТС), вид документа, дата окончания, сколько дней осталось, компания, статус.

Используйте Dashboard для оперативного контроля документов. Расширенный список истекающих документов и предстоящее ТО доступны в разделе **Транспорт → Обслуживание** (см. раздел 11).

## 3. Водители

- **Список водителей** — таблица с поиском и сортировкой.
- **Добавить водителя** — кнопка «Создать» → форма: имя, фамилия, контакты, документы (права, Code 95, медсправка и т.д.), компания, PIN для входа в приложение водителя (4–6 цифр).
- **Карточка водителя** — просмотр всех данных, документов, привязок.
- **Редактирование** — изменение данных и документов.
- При необходимости в карточке/редактировании можно **удалить** водителя (если это предусмотрено интерфейсом).

PIN водителя нужен ему для входа в мобильное приложение (раздел «Водитель»).

---

## 4. Транспорт

### Тягачи

- **Список тягачей** — таблица с основными данными (марка, модель, госномер, статус, компания).
- **Добавить тягач** — форма: марка, модель, госномер, компания, даты документов (техосмотр, страховка, техпаспорт), при необходимости — наличие CAN (для автоматического одометра).
- **Карточка тягача** — полные данные, документы, история.
- **Редактирование** — изменение данных и сроков документов.

### Прицепы

- **Список прицепов** — таблица (марка, госномер, тип, компания и т.д.).
- **Добавить прицеп** — тип (в т.ч. контейнерный), марка, госномер, документы (техосмотр, страховка, TIR, техпаспорт).
- **Карточка** и **редактирование** — по аналогии с тягачами.

Транспорт привязывается к рейсам при создании или редактировании рейса.

### Перевозчики (сторонние / external carriers)

- **Список перевозчиков** (**/carriers**) — справочник сторонних перевозчиков (компании, контакты). Используется при создании рейса с перевозчиком «третья сторона».
- **Добавить перевозчика** — название, контакты, реквизиты.
- **Карточка** и **редактирование** — просмотр и изменение данных.

При выборе перевозчика третьей стороны в форме рейса можно указать **существующего перевозчика из справочника** либо ввести данные тягача/прицепа и фрахт вручную (номера тягачей и прицепов у третьих сторон **не обязаны быть уникальными** в системе).

### Карта

- **Карта** (**/map**) — отображение транспорта на карте (при наличии данных о местоположении).

---

## 5. Клиенты

- **Список клиентов** — компании (клиенты, грузоотправители, грузополучатели).
- **Добавить клиента** — название, страна, город, адрес, реквизиты, контакты.
- **Карточка** и **редактирование** — просмотр и изменение данных.

Клиенты используются в рейсах и заказах как заказчики (customer), грузоотправители (shipper) и грузополучатели (consignee).

---

## 6. Заказы (транспортные заказы)

- **Список заказов** (**/orders**) — таблица транспортных заказов с поиском и фильтрами.
- **Создать заказ** — форма с данными заказа (маршрут, груз, клиент и т.д.).
- **Карточка заказа** — просмотр заказа, связь с рейсом (если заказ уже в рейсе).

**Связь заказ ↔ рейс:**

- **Создать рейс из заказа** — из карточки заказа можно создать новый рейс; заказ автоматически привязывается к рейсу.
- **Добавить к существующему рейсу** — в карточке заказа кнопка «Добавить к рейсу» открывает модальное окно со списком подходящих рейсов (не завершённые). Один заказ может быть привязан только к одному рейсу; добавить можно только если рейс ещё не завершён. После выбора рейса и подтверждения заказ присоединяется к выбранному рейсу (консолидация грузов в одном рейсе).
- **Добавить заказы из карточки рейса** — в просмотре рейса в блоке «Заказы в рейсе» (если рейс не завершён) доступны выбор одного или нескольких доступных заказов и кнопка «Добавить заказы в рейс»; выбранные заказы присоединяются к этому рейсу. Уже привязанный заказ можно убрать из рейса кнопкой «Убрать из рейса».

## 7. Рейсы

### Список рейсов

- **Рейсы** — таблица с фильтрами и сортировкой (по дате, статусу, водителю и т.д.).
- Из списка можно перейти к **созданию** рейса, **просмотру** или **редактированию** выбранного рейса.

### Создание рейса

Пошагово заполняются данные:

1. **Экспедитор** — выбор компании (expeditor) и банка для реквизитов.
2. **Перевозчик**:
   - свой транспорт (экспедитор или другая внутренняя компания);
   - либо **третья сторона (third party)** — выбор **стороннего перевозчика из справочника** (/carriers) или ввод данных вручную; указываются тягач, прицеп и сумма фрахта. Номера тягачей и прицепов третьих сторон могут повторяться (не уникальны в системе). В рейс добавляется расход на оплату субперевозчику.
3. **Транспорт** — водитель, тягач, прицеп (или данные третьей стороны: марка/номер тягача и прицепа, фрахт).
4. **Маршрут (шаги)** — точки погрузки и разгрузки:
   - тип шага: погрузка (loading) / разгрузка (unloading);
   - страна, город, адрес, дата и время;
   - порядок шагов можно менять (drag-and-drop при редактировании).
5. **TIR / таможня** — при международных перевозках можно включить таможенное оформление (TIR) и указать **TIR пограничный пункт** (адрес пункта пропуска/таможни).
6. **Грузы (cargos)** — один или несколько:
   - клиент (customer), грузоотправитель (shipper), грузополучатель (consignee);
   - цена фрахта, НДС;
   - **позиции груза (items)**: описание, упаковки, паллеты, вес (нетто/брутто), объём, погрузочные метры, при необходимости — опасный груз, температура, код ТН ВЭД (customs_code) и т.д.;
   - привязка груза к **шагам погрузки и разгрузки** (какие шаги маршрута относятся к этому грузу).
7. Для контейнерных рейсов — номер контейнера и пломбы (если используются в форме).

После сохранения создаются рейс, шаги маршрута, грузы и позиции; при перевозчике третьей стороны — при необходимости компания, тягач, прицеп и расход. В одном рейсе может быть **несколько заказов/грузов** (консолидация); заказы добавляются из раздела **Заказы** через «Добавить к рейсу», при создании рейса из заказа или в **карточке рейса** в блоке «Заказы в рейсе».

### Просмотр рейса

На странице отображаются:

- **Шапка**: номер рейса, статус, схема (свой/третья сторона), экспедитор, перевозчик, транспорт (водитель, тягач, прицеп), при third party — фиксированная оплата.
- **Сводные показатели**: общий вес, объём, погрузочные метры, фрахт с НДС/без, стоимость товара, сумма по счетам поставщика и т.д.
- **Редактор маршрута** (аккордеон «Маршрут») — компонент **Trip Route Editor**: изменение порядка шагов перетаскиванием, сохранение нового порядка.
- **Заказы в рейсе** — блок со списком заказов, привязанных к рейсу: по каждому заказу — ссылка на карточку заказа и кнопка «Убрать из рейса». Если рейс **не завершён**, ниже отображается список доступных заказов (ещё не привязанных к рейсу); можно выбрать один или несколько заказов и нажать **«Добавить заказы в рейс»** — выбранные заказы присоединяются к рейсу (консолидация). Таким образом добавлять заказы можно и из **карточки рейса**, и из карточки заказа («Добавить к рейсу»).
- **Грузы по клиентам** — группы грузов с развёрнутыми данными:
  - по каждому грузу: отправитель → получатель, упаковки, паллеты, вес, объём, фрахт, стоимость товара, счёт поставщика;
  - точки маршрута, связанные с этим грузом (погрузка/разгрузка), с датой и временем.
- **Документы по грузу**:
  - **CMR** — номер, кнопка «Сгенерировать CMR»: создаётся PDF и привязывается к грузу;
  - **Счёт (Invoice)** — генерация счета по грузу, сохранение PDF.
- **Расходы по рейсу** — список расходов (топливо, платные дороги, парковка и т.д.), при необходимости добавление/редактирование со стороны менеджера (если реализовано в интерфейсе).

Водитель выполняет рейс в мобильном приложении (статусы шагов, одометр, загрузка документов, расходы). Менеджер здесь контролирует маршрут, грузы и документы.

**Публичное отслеживание:** ссылка вида **/track/{token}** позволяет передать клиенту или партнёру доступ к отслеживанию рейса **без входа** в систему (токен привязывается к рейсу).

### Редактирование рейса

Те же блоки, что и при создании: экспедитор, перевозчик, транспорт, шаги маршрута, грузы и позиции. Изменения сохраняются в существующий рейс. Порядок шагов удобнее менять в **просмотре рейса** через редактор маршрута.

## 8. Статистика

### Дашборд владельца (**/stats/owner**)

- **KPI** за выбранный период: рейсы, пробег, выручка (фрахт), **расходы** (в т.ч. дорожные и **обслуживание/ремонты**), прибыль, дебиторская задолженность, простой.
- Блок **«Истекающие документы»** — краткий список документов с истекающим сроком (поиск, сортировка по дате/названию, пагинация). Две колонки: название ТС/водителя и тип документа с датой (например: «Tehniskā apskate · 13.05.2025»).
- Блок **«Gaidāmais apkope»** (предстоящее ТО) — только **свои** тягачи и прицепы (не третьей стороны), у которых задано следующее ТО по дате или по км в ближайшие 30 дней. Отображаются метки **«Pēc datuma»** (по дате) и **«Pēc km»** (по пробегу). Сортировка и поиск, двухколоночный вид: название ТС | тип и дата/пробег.
- Ссылка «Смотреть все» ведёт на страницу **/maintenance**.

### Обзор (**/stats**)

- Таблица по рейсам за выбранный период: даты, водитель, тягач, пробег, **расходы** и т.д.
- **Расходы (Izdevumi)** отображаются с разбивкой: **Ceļa izdevumi** (дорожные расходы по рейсам) и **Apkope** (затраты на ТО/ремонты за период). Итоговая сумма — сумма этих двух составляющих.
- Фильтры по периоду (дата от/до, быстрые диапазоны), поиск, сортировка.
- Используется для анализа рейсов, пробега и структуры расходов.

### События одометра (**/stats/events**)

- Таблица событий одометра: выезд/возврат в гараж, события по шагам маршрута, расходы водителя (топливо, AdBlue) с привязкой к одометру.
- Фильтры: тип события, водитель, тягач, период (дата от/до).
- Полезно для проверки корректности пробега и привязки расходов к километражу.

### Статистика по клиентам (**/stats/clients**)

- Сводка по клиентам за период (объёмы, рейсы, суммы).

### Простой (**/stats/downtime**)

- Учёт и отчётность по простою транспорта.

## 9. Счета (Invoices)

- **Список счетов** — таблица с номерами, датами, суммами, статусом оплаты (оплачен/частично/не оплачен).
- Фильтры: поиск, статус (все / оплачен / частично / не оплачен), сортировка.
- **Открыть PDF** — просмотр или скачивание сгенерированного счета.
- **Внести платёж** — дата и сумма платежа по выбранному счёту; сумма остатка пересчитывается.

Счета по грузам создаются из карточки рейса (кнопка генерации счёта по грузу).

---

## 10. CMR и счета по рейсу

- **CMR** генерируется в **просмотре рейса** по каждому грузу: кнопка «Сгенерировать CMR» → создаётся PDF, сохраняется и привязывается к грузу. Номер CMR можно задать/редактировать в форме.
- **Счёт (Invoice)** по грузу генерируется там же: кнопка генерации счёта → создаётся PDF счёта, запись появляется в разделе **Счета**. Дальнейшая работа с оплатой — в разделе «Счета».

---

## 11. Обслуживание парка (Maintenance)

Раздел **Транспорт → Обслуживание** (**/maintenance**) объединяет контроль документов и предстоящего ТО.

### Истекающие документы

- На странице **/maintenance** отображается полный список **всех истекающих документов** (водители, тягачи, прицепы): техосмотр, страховка, права, Code 95, TIR, техпаспорт, медсправка и т.д.
- Доступны **поиск**, **сортировка** (по дате, по названию), **пагинация** и выбор количества строк на странице.
- На десктопе — таблица, на мобильных — карточки.

### Предстоящее ТО (по дате и по пробегу)

- Блок **«Gaidāmais apkope»** показывает тягачи и прицепы **вашей компании** (не третьей стороны), у которых задано следующее ТО по дате или по км в ближайшие 30 дней.
- Для каждой позиции доступна кнопка **«Провести ТО»** — переход к форме создания записи о проведённом обслуживании с уже подставленным транспортным средством.
- Сортировка и поиск по списку.

### Журнал записей ТО

- **Журнал записей** (**/maintenance/records**) — список всех записей о проведённом ТО/ремонтах: транспорт (марка, номер), дата, одометр, описание, **стоимость**.
- Вверху отображается **сумма затрат** по выбранным фильтрам (период, поиск). Сумма пересчитывается при изменении периода или поиска.
- Фильтры: **период** (дата от / до), **поиск** по названию транспорта. Сортировка: **по дате** или **по названию транспорта**.
- Клик по строке открывает карточку записи с деталями.

### Создание и редактирование записи ТО

- **Создать запись** (**/maintenance/records/create**) — форма: выбор **тягача или прицепа** (обязательно одно из двух), дата проведения, одометр (при наличии), описание работ, стоимость (необязательно, если работы своими силами).
- **Следующее ТО**: обязательно заполнить **одно из полей** — «Дата следующего обслуживания» или «Пробег следующего ТО (км)». Эти значения сохраняются в карточке транспортного средства и отображаются в блоке предстоящего ТО и в карточке тягача/прицепа.
- **Редактирование** записи — те же поля; при сохранении обновляются дата/пробег следующего ТО на выбранном ТС.

Стоимость ремонтов и ТО учитывается в **финансовых показателях** (дашборд владельца, статистика по рейсам — см. ниже).

---

## 12. Профиль и выход

- **Профиль** — просмотр и при необходимости изменение своих данных (если реализовано).
- **Выйти** — кнопка в меню в шапке справа; завершает сессию и перенаправляет на страницу входа.

## Краткая памятка по разделам

| Раздел | Основные действия |
|--------|--------------------|
| **Dashboard** | Истекающие документы водителей, тягачей, прицепов (поиск, сортировка) |
| **Водители** | Список, создание, карточка, редактирование, PIN для приложения |
| **Тягачи / Прицепы** | Список, создание, карточка, редактирование, документы, следующее ТО по дате/км |
| **Перевозчики** | Справочник сторонних перевозчиков; выбор при создании рейса «третья сторона» |
| **Обслуживание** | Истекающие документы (все типы), предстоящее ТО, журнал записей ТО, форма записи (дата, одометр, описание, стоимость, следующее ТО) |
| **Заказы** | Список заказов; создание рейса из заказа; добавление к рейсу из карточки заказа или из карточки рейса (блок «Заказы в рейсе») |
| **Рейсы** | Список, создание, просмотр (маршрут, **заказы в рейсе** — добавить/убрать заказы, грузы, CMR, счета, расходы), редактирование; несколько заказов в одном рейсе |
| **Статистика → Дашборд владельца** | KPI, истекающие документы, предстоящее ТО (свои ТС), разбивка расходов (дорога + апкope) |
| **Статистика → Обзор** | Рейсы за период, пробег, расходы (ceļa izdevumi + apkope) |
| **Статистика → События / Клиенты / Простой** | События одометра, статистика по клиентам, простой |
| **Счета** | Список счетов, открытие PDF, внесение платежей |
| **Отслеживание** | Публичная ссылка /track/{token} для просмотра рейса без входа |

## Связь с приложением водителя

- Водитель входит в **мобильное приложение** по **PIN** (назначается в карточке водителя).
- В приложении водитель: отмечает выезд/возврат в гараж с одометром, меняет статусы шагов (погрузка/разгрузка), загружает документы по шагам, добавляет расходы по рейсу.
- Менеджер в веб‑интерфейсе видит рейсы, маршрут, грузы, сгенерированные CMR и счета, расходы и события одометра в статистике. При необходимости маршрут можно скорректировать (редактор маршрута в просмотре рейса).

## Мобильная версия

- Интерфейс администратора адаптирован под мобильные устройства: боковое меню открывается по кнопке «гамбургер», таблицы на малых экранах отображаются в виде карточек, формы и кнопки имеют достаточный размер для нажатия. Страницы Обслуживание, Дашборд владельца, Заказы (модальное окно «Добавить к рейсу») и остальные разделы корректно работают на смартфонах и планшетах.
